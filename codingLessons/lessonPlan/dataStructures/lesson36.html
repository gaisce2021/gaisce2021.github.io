Questions

What data structure will we look at today?

A deque, pronounced "deck", is a double-ended queue. It is a linear data structure where the insertion and deletion operations are performed from both ends. It does not follow the FIFO rule (First In First Out) like a queue. A deque is a generalised version of the queue.

How is a deque implemented? Is it similar to a queue?

A deque has some properties in common with a queue and stack. It is a linear data structure like a queue and stack. However, it is more general than a stack or queue because it has some properties in common with a deck of cards.

What are the methods of the deque?

A deque supports element insertion and removal at both ends. It has methods to access the elements at both ends of the deque. Methods are provided to insert, remove, and examine the element. We can also check if the deque is empty and its size.

Tell me the space and time complexity of the deque.

The time and space complexity of deque methods depend on the implementation. In Python, the deque class is implemented using a doubly-linked list. The append and pop operations have O(1) time complexity. The append_left and pop_left operations also have O(1) time complexity. The space complexity of a deque is O(n), where n is the number of elements in the deque.

Code Examples

We will base this implementation on the DLL. We will have a Node class with previous and next pointers. We have two insert methods, two delete methods, a method to print the deque as a list, a method to check the size of the deque and a method to check if the deque is empty. 

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None
        self.prev = None


class Deque:
    def __init__(self):
        self.front = None
        self.rear = None

    def insert_front(self, data):
        new_node = Node(data)
        if self.front is None:
            self.front = new_node
            self.rear = new_node
        else:
            new_node.next = self.front
            self.front.prev = new_node
            self.front = new_node

    def insert_rear(self, data):
        new_node = Node(data)
        if self.rear is None:
            self.front = new_node
            self.rear = new_node
        else:
            new_node.prev = self.rear
            self.rear.next = new_node
            self.rear = new_node

    def delete_front(self):
        if self.front is None:
            return None
        elif self.front == self.rear:
            temp = self.front
            self.front = None
            self.rear = None
            return temp.data
        else:
            temp = self.front
            self.front = temp.next
            temp.next.prev = None
            temp.next = None
            return temp.data

    def delete_rear(self):
        if self.rear is None:
            return None
        elif self.front == self.rear:
            temp = self.rear
            self.front = None
            self.rear = None
            return temp.data
        else:
            temp = self.rear
            self.rear = temp.prev
            temp.prev.next = None
            temp.prev = None
            return temp.data

    def is_empty(self):
        return self.front is None

    def size(self):
        count = 0
        cur_node = self.front
        while cur_node is not None:
            count += 1
            cur_node = cur_node.next
        return count

    def print_deque(self):
        output = []
        cur_node = self.front
        while cur_node is not None:
            output += [cur_node.data]
            cur_node = cur_node.next
        print(output)
