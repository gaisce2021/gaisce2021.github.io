This week we will take at the priority queue. If you can recall lesson 29 about queues then we will build something similar today. It may help you to revisit this data structure before looking at the slight difference between a standard FIFO and a priority queue (PQ).

How are queues and priority queues different?

A queue is a linear data structure that follows the FIFO principle as mentioned above. In a queue, the first item that is added to the queue is the first item that is removed from the queue. A priority queue is a type of queue where each item has a priority value associated with it. The items in the priority queue are ordered based on their priority values, and the item with the highest priority value is always at the front of the queue. When an item is removed from the priority queue, it is always the item with the highest priority value that is removed. We will use the same methods in the PQ as in the FIFO queue. The same methods are used in both queues.

Tell me the space & time complexity of each method.

We will begin by taking look at the time complexity before looking at the space complexity.
The time complexity of the enqueue method is O(n) in the worst case because it has to traverse the entire list to find the correct position to insert the new item. The dequeue method has a time complexity of O(1) because it removes and returns the first item in the list. The peek method also has a time complexity of O(1) because it returns the first item in the list without removing it. The isEmpty method has a time complexity of O(1) because it checks if the length of the list is zero. The size method also has a time complexity of O(1) because it returns the length of the list.
The space complexity of this implementation is O(n) because it uses a list to store the elements.

Where would a priority queue be used in real life?

We will look at three real world examples where a priority queue would be used. The general idea is to identify a reason why someone or something should be prioritised and order that queue accordingly. This can be quantified by time, distance, exam grades to receive a college place, etc.

  1. Hospital emergency queue: In this queue of patients, the patient with the most critical situation is the first in a queue, and the patient who doesn't need immediate medical attention will be the last. The hospital staff will treat the person who has been in a serious car accident before treating a child with a sprained wrist.
  2. Another example of a priority queue is the air traffic control system. In this system, the planes are assigned priorities based on their urgency and importance. The planes with higher priority are given clearance to land or take off first. A plane that is low on fuel may be given a higher priority than other planes which have enough fuel to remain in the air for another 60 minutes.
  3. Finding the closest restaurants/hotels/bathrooms to a given location. Would a jogger in New York City be more interested in the nearest toilet in Manhattan or in Los Angeles?

The methods are enqueue to add an item to the back of the queue, dequeue to remove an item from the front of the queue, isEmpty to check if the queue is empty, peek to look at the item at the top of the queue, and size to see what the length of the queue is. class Queue:
    def __init__(self):
        self.queue = []

    def enqueue(self, item):
        self.queue.append(item)

    def dequeue(self):
        if not self.isEmpty():
            return self.queue.pop(0)

    def isEmpty(self):
        return len(self.queue) == 0

    def peek(self):
        if not self.isEmpty():
            return self.queue[0]

    def size(self):
        return len(self.queue)

Now that we have revised the standard FIFO queue, we can take a look at today's priority queue. The main difference you will see is in the enqueue method. It prioritises new elements and can move them to the front of the list if they are deemed more important. 

class PriorityQueue:
    def __init__(self):
        self.queue = []

    def enqueue(self, item, priority):
        inserted = False
        for i in range(len(self.queue)):
            if priority < self.queue[i][1]:
                self.queue.insert(i, (item, priority))
                inserted = True
                break
        if not inserted:
            self.queue.append((item, priority))

    def dequeue(self):
        return self.queue.pop(0)[0]

    def isEmpty(self):
        return len(self.queue) == 0

    def peek(self):
        return self.queue[0][0]

    def size(self):
        return len(self.queue)

class MusicPlayer:
    def __init__(self):
        self.dll = DLL()

    def add_song(self, song):
        self.dll.insert(song)

    def play(self):
        if self.dll.head is None:
            print("Playlist is empty.")
            return
        print("Playing:", self.dll.head.data.title,
                "by", self.dll.head.data.artist)
        self.dll.head = self.dll.head.next

    def skip(self):
        if self.dll.head is None:
            print("Playlist is empty.")
            return
        if self.dll.head.next is None:
            print("End of playlist.")
            return
        if self.dll.head.next is not None:
            print("Skipping:", self.dll.head.data.title,
                "by", self.dll.head.data.artist)
            self.dll.head = self.dll.head.next
            self.dll.head.next.prev = self.dll.head

    def traverse(self):
        self.dll.traverse()
