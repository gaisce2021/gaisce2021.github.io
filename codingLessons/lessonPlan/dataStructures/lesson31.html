Questions

Can you explain the difference between this linked list and the one from last week?

We will look at a doubly linked list (DLL) today. he main difference between the SLL and the DLL is that the SLL contains only one pointer to traverse through the list while a DLL contains two pointers to traverse through the list. In SLLs, the traversal can be done using the next node link only. This means traversal is possible in one direction only (only from start to end). In a DLL, the traversal can be done using the previous node link or the next node link. This means you can traverse in both directions (forward and backward). The SLL occupies less memory than DLL on account of having less pointers.

Tell me the space and time complexity of the DLL.

We will take a look at the time complexity of each method first. Insertion: O(1) if you know the index you need to insert at. If you do not, you have to iterate over all elements until you find the one you want. Deletion: O(1) if you know the index of the element you want to remove. If you do not, you have to iterate over all elements until you find the one you want. Search: O(n) as it requires iterating over all elements in the list. Your desired item may be at the back of the list. Traversal: O(n) as it requires iterating over all elements in the list. This requires you to look at each element of the list as there are n-many items in the list.

The space complexity of each method is easier to understand. It is O(n) as it requires storing n elements in memory.

Why should I go to the effort of making a DLL instead of sticking with the SLL?

Doubly linked lists allow for bidirectional traversal, which is not possible with singly linked lists. DLLs make it easier to reverse the list because you can traverse it in both directions.
Doubly linked lists make it easier to delete elements from the list because you do not need to know the element before the one you want to delete.

Code Examples

We will take the code from the SLL from last week and highlight the amendments in bold. 

class Node:
    def __init__(self, data):
        self.data = data
        self.next = None
        self.prev = None

class DLL:
    def __init__(self):
        self.head = None

    def insert(self, data):
        new_node = Node(data)
        if self.head is None:
            self.head = new_node
            return
        cur_node = self.head
        while cur_node.next:
            cur_node = cur_node.next
        cur_node.next = new_node
        new_node.prev = cur_node

    def delete(self, key):
        cur_node = self.head
        if cur_node and cur_node.data == key:
            self.head = cur_node.next
            self.head.prev = None
            cur_node = None
            return
        while cur_node and cur_node.data != key:
            cur_node = cur_node.next
            if cur_node is None:
                return
            if cur_node.next is None:
                cur_node.prev.next = None
                return
            cur_node.prev.next = cur_node.next
            cur_node.next.prev = cur_node.prev

    def search(self, key):
        cur_node = self.head
        while cur_node:
            if cur_node.data == key:
                return True
            cur_node = cur_node.next
        return False

    def traverse(self):
        if self.head is None:
            print("List is empty")
            return

        cur_node = self.head
        while cur_node:
            print(cur_node.data, end="")
            if cur_node.next:
                print(" -> ", end="")
            last = cur_node
            cur_node = cur_node.next

        print("\nTraversal in reverse direction")
        while last:
            print(last.data, end="")
            if last.prev:
                print(" -> ", end="")
            last = last.prev

Now let's discuss the amended code. The Node class has an additional previous pointer. This is where the D in doubly linked list comes from. The remaining changes are in the DLL class.

Search is the easiest method to write. Why? Because it's unchanged. Traverse keeps its initial method to traverse the tree going forward but then adds code to start at the tail of the list and work backwards, traversing the list from the end. Delete makes amendments to the previous pointers so something deleted in the middle of the list can no longer be pointed to by the item before or after it. It is important to reassign pointers of neighbouring items as they may cause issues if you call the traverse method at a later point.
