Questions

What is the first data structure we will look at today?

We will start off with the stack. This follows the principle of last-in, first-out (LIFO). It may sometimes be called FILO (first in, last out). This means that the last element that is added to the stack is the first one that is removed from it. It may help you to think about a stack of books. You'll take the top book off the stack first and work your way down to the one on the floor.

What does a stack look like in code?

A stack can be implemented using an array or a linked list, and it supports two basic operations: push and pop. Push adds a new element to the top of the stack, and pop removes and returns the top element from the stack. Stacks are useful for many applications, such as reversing a string, evaluating expressions, or implementing recursion. We won't look at the linked list implementation until we encounter linked lists later in the course.

Will we add any other methods?

We have push and pop as mentioned above. We will also look at three additional methods.

  • Push: This operation adds an element to the top of the stack.
  • Pop: This operation removes and returns the element at the top of the stack.
  • Top: This operation returns the element at the top of the stack without removing it.
  • IsEmpty: This operation checks if the stack is empty or not.
  • Size: This operation returns the number of elements in the stack.

What is the time complexity of the operations in the stack?

The time complexity of pop, top, isEmpty, and size is O(1). The best case scenario of push is O(1) but the worst case scenario of push is O(n). This depends on how the stack is built and how exactly you're pushing a new item onto the stack.

What is the space complexity of the operations in the stack?

The space complexity of push, pop, top, isEmpty, and size is O(1). This is great! This means that the amount of memory required to perform these operations does not depend on the size of the stack. The stack is not rearranged in any operations which makes it less intensive for the computer.
