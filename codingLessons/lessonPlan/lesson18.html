Questions

What is Abstraction?

Abstraction is the fourth principle of OOP. So far we have looked at inheritance, encapsulation, and polymorphism. Abstraction may seem similar to encapsulation but there is a subtle difference. Encapsulation involves blocking direct access to the data by the user. Abstraction hides unnecessary information from the user. Why have a user interact directly with parent classes such as Animal and Vehicle when they can interact with Cat or Toyota?

How does abstraction work?

Abstract classes typically have a methods without a body. This is because these classes are only ever intended to be used as a base class. They are not intended to be used or called. These classes are blueprints which allow child classes fill in the fine details.

What's next after learning about these four principles?

We will try use OOP to build the foundation for gaming. We will try to create basic games over the next few weeks which use objects and interact with each other. This is a classic use case for OOP and will definitely be one of the highlights of our lessons. This gives you a chance to build up a portfolio of work you've created down through this course when applying for college courses and for jobs.

Code Examples

Lets try our hand at coding some abstraction. We can build on last week's lesson about animals. 

from abc import ABC, abstractmethod


class Animal(ABC):
    def __init__(self, name, legs):
        self.name = name
        self.legs = legs

    @abstractmethod
    def make_noise(self):
        raise NotImplementedError("Subclass must implement this method")

    @abstractmethod
    def eat_food(self):
        raise NotImplementedError("Subclass must implement this method")
