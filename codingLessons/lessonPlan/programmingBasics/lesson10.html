Dictionaries

So now we've seen a few different data types such as lists, strings and integers. The dictionary is another way you can store information. This structure functions similarly to the physical dictionary in real life. The Python dictionary takes in two things: a key and a value. A key is unique but two keys can share a value. If it helps to think about the physical dictionary; you might have a key computer with the value an electronic device for storing and processing data, typically in binary form, according to instructions given to it in a variable program. The word computer here is unique - open up a physical dictionary, and you'll find each word listed in alphabetic order only appearing once. Each key has a value paired with it. In the case of the physical dictionary, the value paired with each word is a definition.

Code Example

dictionary = {"key1": "value1"}
print(dictionary.__len__())
dictionary["key2"] = "value2"
print(dictionary.keys())
print(dictionary.values())
print(dictionary)
dictionary.pop("key1")
print(dictionary.__len__())
print(dictionary)

dict_values(['value1', 'value2'])

1
{'key2': 'value2'}

Confused? Let's take a look at the code line by line and see what output it has and why. The first line creates a dictionary and populates it with one key and one value. The next line prints the length of the dictionary. We only have one item in the dictionary, so it has a length of 1. The next line creates a new key called key2 with a value of value2 in our dictionary. The following line shows you the keys in your dictionary and the line after shows you the values in your dictionary. The next line simply prints the keys and values of the dictionary. The following line finds key1 in the dictionary and removes the entry from the dictionary. The next line prints the length which, again, is 1. The last line prints out the dictionary which gives us our dictionary with one entry {'key2': 'value2'}.
