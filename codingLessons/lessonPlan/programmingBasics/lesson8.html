Print vs Return

I will now introduce the return statement. It is typically used in place of the print function in the functions that we have created already. Printing shows a particular value at a particular point in time. Returning assigns a value to your function. If you run two functions, one with a print statement and one with a return statement something interesting happens. Let's look below. 

def print_function(year_of_birth):
    current_year = 2022
    print(current_year - year_of_birth)

def return_function(year_of_birth):
    current_year = 2022
    return current_year - year_of_birth

print(print_function(1950))
print(return_function(1950))



72
None


72

Notice that None appears? This is because when a function does not have a return statement, its return value is defaults to None. So the next question is: why do we care what a function returns? After all both functions show the number 72, what's the big deal? Well let's extend the above code example below, and we'll find out why you should care. 

return_example = return_function(1950)
print(return_example - 20)

print_example = print_function(1950)
print(print_example - 20)

52
72
Traceback (most recent call last):
File "lesson8.py", line 13, in module
print(print_example - 20)
TypeError: unsupported operand type(s) for -: 'NoneType' and 'int'

We save both function outputs to variables called return_example and print_example. So here we try take 20 away from 72. We expect that both values should give us an answer of 52. We first take away 20 from return_example and get 52. Great! No problems here. Let's try the same thing with print_example now. Oh no, we broke our code. The code prints out 72 from the print function before a red error message pops up. Welcome to the joys of programming - finding an error and figuring out what just happened. Let's walk through the error message line by line to see what went wrong and how we can fix it. So the Traceback line traces back through the line of errors in the stack. In this case only one part of the code is involved in the error, so we'll look at the stack a little later. Next the error message gives you the file path and the specific line that caused the issue. We can see the file on my machine is called lesson8.py and line 13 is the place where the code broke. The following line of the error message is print(print_example - 20). This is the specific code that Python is telling you it can't run. The final line is telling us the specific details of the error. It was a type error - there was some issue between the data types involved in the code. It tells us that we tried to use subtraction which wasn't allowed. But 72 and 20 are both numbers, why can't we subtract two numbers? The error message tells us that 20 was an integer but that the value of print_example was None which is not an integer. We tried to perform an operation that was incompatible between two types and this caused our code to break.

Lesson 7 Solution

hobbies_list = ['walking', 'programming', 'reading', 'taekwondo']
def print_while(hobbies):
    hobby = 0
    while hobby < len(hobbies):
        print(hobbies[hobby])
        hobby += 1

def print_for(hobbies):
    for index, hobby in enumerate(hobbies):
        output = (index, hobby)
        print(output)

print_while(hobbies_list)
print_for(hobbies_list)
