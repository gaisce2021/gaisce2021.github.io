Numeric Data Types

As discussed in lesson 2, we will look at two types of numbers in this course. These are integers and floats. Put simply, integers are whole numbers and floats are decimal numbers. Let's look at some examples below.

Integers

Maybe it's your birthday today. In that case, happy birthday! If not then bookmark this page and come back here on the day of your birthday. If you're born on February 29th please enjoy your wait. So what happens when your birthday comes around? The day before your birthday you might say "I'm 20 years old". The next morning you'll wake up and, if you go to the cashier in a local shop for wine, you'll probably show some form of ID and say "I'm 21 years old". So let's look at this in code. Here we'll show the initial age increased by one. This is your new age when your birthday comes around each year. The first print statement will show you that you're 21 today. The second print statement uses a comparison and will give us a Boolean (True/False) answer. This checks that your previous age plus one is equal to your new age. Try changing the age_yesterday variable from 19 to 20 and running the code on your end. It should read False now. Why? Because 19 and 1 don't add up to 21. 

age_yesterday = 20
birthday_age_change = 1
age_today = 21
print(age_yesterday + birthday_age_change)
print(age_yesterday + birthday_age_change == age_today)

Functions

One thing you will see lots of throughout your program career is functions. You define a function using def from last week's list of keywords. Programmers use functions because you define a process once, then you can call a function multiple times with different values to repeat that process. Let's define a function and add comments to explain the process. This function is called calculate_wages. This is a meaningful name as it lets other programmers know you're trying to calculate someone's wages. This function has three parameters. The first parameter is base_hours. This tells you how many hours someone is supposed to work per week according to their contract. The next parameter is hours_overtime which is the additional hours worked that week on top of contracted hours. The last parameter is hourly_rate which tells you how much this person is paid per hour worked. A parameter is an argument passed to a function. Passing different arguments to the same function can yield different results. Let's look at an example of that.
Our first employee works a 40-hour week. They are paid €15 an hour. They worked 5 hours overtime on the week in question. The second employee works 20 hours each week. They are paid €12 per hour. This employee did not work any overtime this week. The first employee earns €712.50 this week. The second employee earns €240.00 this week. 

# Function to calculate a person's weekly pay
def calculate_wages(base_hours, hours_overtime, hourly_rate):
    # Calculate pay received for regular hours
    base_pay = hourly_rate * base_hours
    # Now calculate overtime pay at time and a half
    overtime_pay = (hourly_rate * 1.5) * hours_overtime
    # Add base and overtime wages
    gross_pay = base_pay + overtime_pay
    # Print out the wages before tax
    print(gross_pay)

calculate_wages(40, 5, 15)
calculate_wages(20, 0, 12)












712.5
240.0
