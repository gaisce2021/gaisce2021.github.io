What is Encapsulation?

Let's look at the first website on Google for a definition of encapsulation. Encapsulation is a way to restrict the direct access to some components of an object, so users cannot access state values for all of the variables of a particular object. Encapsulation can be used to hide both data members and data functions or methods associated with an instantiated class or object. This definition comes directly from the Sumo Logic website. Encapsulation allows you to make certain parts of classes read only or write only. Maybe you want someone to be able to see a piece of information but not directly alter it. Using encapsulation can prevent unauthorised users from directly tampering with the code.

Why would you use encapsulation?

As mentioned above, you may not want someone to directly tamper with data. Perhaps you want them to only be able to read data. Sometimes you may want someone to update some attributes by themselves. This is where getters and setters come into play. Getters and setters are commonly used in OOP. As their names suggest, they are used to get and set data. We'll look at them in a little more detail below in the coding section.

Code Examples

Let's review the getters and setters mentioned above. We will take the example of the Vehicle subclass, the Bus class from last week. It makes sense to be able to check the number of seats for the purposes of the license held by the bus driver using a getter. We should use getters and setters for the availability of seats. Maybe a number of seats are unavailable, and it might be quicker set the available seating capacity rather than adding passengers one by one. Likewise, if a number of passengers get off at a particular stop, it might be easier to increase the available number of seats by a dozen in one go. 

class Vehicle:
  def __init__(self, colour, license_code, number_plate):
    self.colour = colour
    self.license_code = license_code
    self.number_plate = number_plate

  def drive(self):
    print("This vehicle is in motion.")


class Bus(Vehicle):
  def __init__(self, colour, number_plate):
    super().__init__(colour, "D", number_plate)
    self._seats = 40
    self._availability = 40

  def add_passenger(self):
    if 0 <= self.availability < self.seats:
        self.availability -= 1
        print("Total number of passengers aboard: " + str(self.seats - self.availability))
    else:
        print("No seats available, wait for the next bus.")

  def remove_passenger(self):
    if 0 <= self.availability < self.seats:
        self.availability += 1
        print("Total number of passengers aboard: " + str(self.seats - self.availability))
    else:
        print("Bus empty!")

  def seats(self):
    return self._seats

  def availability(self):
    return self._availability

  def availability(self, capacity):
    if capacity > 0 and capacity <= self.seats:
      self._availability = capacity
    else:
      print("Invalid number of seats.")

class Siamese(Cat):

    super().__init__(name, age, colour)
    self.eye_colour = "Blue"

  def describe(self):
    print(self.name + " has " + self.eye_colour + " eyes and " + self.colour " fur.")

class Bengal(Cat):
  def __init__(self, name, age, colour):
    super().__init__(name, age, colour)
    self.pattern = "Spotted"

  def eat(self):
    print(self.name + " is eating.")

lucky = Siamese("Lucky", 3, "white")
lucky.describe()
